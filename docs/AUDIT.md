# Audit checklist

This document is for an external reviewer evaluating the codebase end to end.

## 1. Build provenance

- Toolchain pinned via `rust-toolchain.toml` (`channel = "stable"`).
- All deterministic outputs are regenerated by `tea-bsv reproduce` and diffed against committed copies under `vectors/`.

## 2. Hash and curve primitives

| Item | Where | Test |
|---|---|---|
| BSV double-SHA256 | `crates/bsv/src/lib.rs::double_sha256` | known-vector test for the empty input |
| BSV curve arithmetic | `crates/bsvcurve/src/lib.rs` (via `k256`) | scalar add, point compress/decompress, ECDH agreement |
| ECDSA low-S RFC 6979 deterministic | `crates/bsvcurve/src/lib.rs::ecdsa_sign_prehash` | sign+verify round-trip; low-S canonicalisation asserted |
| HKDF-SHA-256 (single-block expand) | `crates/bsvcurve/src/lib.rs::hkdf_extract` / `hkdf_expand_one_block` | round-trip test |

## 3. Protocol primitives

| Item | Where | Test |
|---|---|---|
| Sub-key derivation | `crates/tea/src/lib.rs::derive_subkey` | deterministic, equal across two runs |
| Key material derivation | `crates/tea/src/lib.rs::derive_key_material` | two-party agreement on `S`, `K_master`, `L_inv`, `L_pay` |
| Per-field key + commitment | `crates/tea/src/lib.rs::commit_one` | recomputation matches |
| Note body + signature | `crates/tea/src/lib.rs::sign_note` / `verify_note` | sign + verify + tamper-reject |

| Item | Where | Test |

## 9. CLI

| Subcommand | What it does |
|---|---|
| `tea-bsv selftest` | All five primitives end-to-end |
| `tea-bsv reproduce` | Regenerate and diff every committed vector |
| `tea-bsv worked-example` | Print the canonical worked-example JSON |
| `tea-bsv anchor` | Build a batch over JSON-supplied notes |
| `tea-bsv prove` | Build an inclusion bundle for one note |
| `tea-bsv verify` | Verify an inclusion bundle |
| `tea-bsv query` | Exercise the proofstore retrieval flow |
| `tea-bsv disclose` | Issue a scoped disclosure envelope |
