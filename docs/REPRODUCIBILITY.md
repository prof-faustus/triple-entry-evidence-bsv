# Reproducibility

## Vectors

| Vector | Source | Regenerated by |
|---|---|---|
| `vectors/merkle/bsv_block_v1.json` | Real BSV mainnet block (two txids; canonical Merkle root) | `tea-bsv reproduce` (recomputed via `merkle_root_of_leaves`) |
| `vectors/tea/worked_example_v1.json` | TEA worked example on the BSV curve, fixed master scalars `0x11…11` / `0x22…22` | `tea-bsv worked-example` (printed) and `tea-bsv reproduce` (diffed) |
| `vectors/study/simstudy_v1.json` | Synthetic invoice population, `SEED = 2_026_053_003`, M = 200, inclusion sample = 64, selective sample = 64 | `tea-bsv-simstudy -m 200 --inclusion-sample 64 --selective-sample 64 --vector-out vectors/study/simstudy_v1.json` |

## How `reproduce` works

`tea-bsv reproduce` reads each committed vector, recomputes the same value from the source code, and diffs. Any drift returns a non-zero exit. CI should run `cargo build --release && cargo test --workspace && cargo run --release -p tee-cli -- reproduce && cargo run --release -p tee-simstudy -- -m 200 --vector-out /tmp/simstudy_check.json` and diff `/tmp/simstudy_check.json` against the committed `vectors/study/simstudy_v1.json`.

## Determinism

- ECDSA uses RFC 6979 deterministic nonce generation with low-S canonicalisation enforced at the API surface. The same prehash and key produce a byte-identical signature on every host.
- HKDF-SHA-256 produces byte-identical output for the same `(salt, ikm, info)`.
- Simulation uses `ChaCha20Rng` seeded with `SEED = 2_026_053_003`; populations and fault outcomes are byte-identical across runs.

## What is NOT bit-deterministic

- Wall-clock timings (informational only; not committed).
- Test ordering (governed by Cargo).
